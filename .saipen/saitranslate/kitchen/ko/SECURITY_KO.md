# 보안 정책

## 범위

SAIPEN은 사양(specification)과 함께 소수의 로컬 설치/내보내기 스크립트(`bootstrap/inject.ps1`/`.sh`, `uninstall.ps1`/`.sh`, `export.ps1`/`.sh`)의 집합입니다. 서버를 실행하지 않으며, 원격 분석(telemetry)을 수집하지 않고, 어떠한 데이터도 외부로 전송하지 않습니다. 스크립트가 수행하는 모든 작업은 사용자가 이미 제어하고 있는 파일(자신의 `~/.claude`, `~/.gemini`, 프로젝트 `.saipen/` 등)에 대한 로컬 파일 시스템 쓰기 작업뿐이며, 첫 수정 전에 자동으로 `.bak` 백업본이 생성되어 보호됩니다.

실제로 보안 보고가 가치 있는 두 가지 경우는 다음과 같습니다:
1. 부트스트랩 스크립트가 주석이나 README에 명시된 범위를 넘어 파일 시스템이나 git 히스토리에 무언가를 수행하는 경우.
2. 프로토콜 자체의 비밀 정보 관리 규칙(RFC.md § 1.1 -- API 키, 토큰, 비밀번호를 `STATE.md`/`BOARD.md`/`LOG.md`/`KNOWLEDGE/`/`kitchen/`에 절대 쓰지 말 것)에 실제 구멍이 있어, SAIPEN을 따르는 에이전트가 비밀 정보를 커밋되는 파일에 유출하게 되는 경우.

## 지원 버전

`main` 브랜치의 최신 태그된 릴리스만 지원됩니다. 이것은 프로토콜 사양이며 수명이 긴 서비스가 아닙니다 -- LTS 브랜치는 없습니다.
